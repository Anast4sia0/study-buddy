# Project Description

## Project Title

StudyBuddy – Student Study Management Application

## Introduction

StudyBuddy is a web application made for students who want to organize their study work in one place. During the semester, students often have many assignments, projects, and exams, which makes it difficult to keep everything organized. Because of this, we decided to create an application that helps users plan their study schedule, manage tasks, and track their progress.

The application was built using React, JavaScript, HTML, CSS, and Local Storage. Instead of using a database, all information is saved in the browser, so users can continue using the application after refreshing the page.


## Project Objectives

The main goals of this project were:

* Make it easier for students to organize their study plans.
* Help users keep track of their assignments and completed work.
* Include a timer to help students stay focused while studying.
* Build a simple and easy-to-use interface.
* Practice React concepts learned during the course.


## Main Features

### User Authentication

The application allows users to create an account and log in. Input validation is included to prevent incorrect information from being entered. Users can also log out at any time.

### Study Plans

Users can create multiple study plans for different subjects. Each study plan can be opened to view its tasks, and plans can also be removed if they are no longer needed.

### Task Management

Inside every study plan, users can add new tasks, choose a priority level, mark tasks as completed, or delete them.
### Subtasks

Large tasks can be divided into smaller subtasks. This helps users complete work step by step. Subtasks can also be marked as completed or deleted.

### Progress Tracking

The application automatically calculates study progress based on completed tasks and subtasks. It also shows simple statistics that help users see how much work has already been finished.

### Focus Timer

A built-in timer is available for study sessions. Users can start, pause, and reset the timer, and different durations can be selected depending on the task.

### Profile and Settings
The application includes a profile page and a settings page. Users can change timer preferences, enable dark mode, and save all settings using Local Storage.

## Development Process

We started by planning what features the application should have and then created the basic structure of the project. First, we implemented task management because it is the main part of the application. After that, we gradually added other features such as user authentication, study plans, subtasks, progress tracking, timer, profile page, settings, and dark mode.
During development, we tested each feature after implementing it, fixed bugs when they appeared, and made small improvements to the interface. GitHub was used to manage the project and keep track of changes.
